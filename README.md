# 펜타힐즈 더블유 3D 일조 · 조망 시뮬레이터

경북 경산 중산지구 **펜타힐즈 더블유** 단지를 공식 배치도·동호수 배치도에서 좌표를 뽑아
3D로 세운 뒤, **어느 세대에 해가 얼마나 드는지**와 **창밖으로 뭐가 보이는지**를 보는 도구입니다.

## 어떤 자료로 만들었나

| 자료 | 출처 | 뽑아낸 것 |
|---|---|---|
| 단지 배치도 | `pentahillsw.co.kr/sub/db.jpg` | 동 위치·방향, 나침반(정북), 축척, 호수·지형·수목 |
| 동·호수 배치도 | `pentahillsw.co.kr/sub/dh.jpg` | 동별 최고 층수, 라인별 주택형 |

배치도를 눈으로 보고 옮긴 게 아니라 **픽셀을 분석해서 계산**했습니다.

- **동 위치·방향** — 배치도 범례의 주택형 색상을 기준으로 십자형 날개 4개를 색으로 찾아내고,
  그 무게중심으로 동 중심을, 4중 대칭 성분으로 십자 회전각을 구했습니다.
- **축척** — 배치도에 표기된 "펜타힐즈1로 20m" 도로 폭을 5곳에서 재서 **1px = 0.711m**.
- **정북** — 배치도 우상단 나침반의 원과 바늘을 픽셀로 검출한 결과
  **정북 = 이미지 위쪽에서 시계방향 24°**. 이 값이 그대로 단지 회전값으로 들어갑니다.
- **지형** — 배치도 픽셀을 물·수목·잔디·아스팔트·보도·모래로 분류해서 다시 칠했습니다.
  화면에 깔린 바닥은 **배치도 사진이 아니라 분류 결과로 새로 그린 지형**입니다.
- **수목** — 수목으로 분류된 영역에서 **1,678그루**를 뽑아 3D로 심었습니다.

### 위도 (일조 계산의 핵심)

경산 중산못 기준 **북위 35.845°, 동경 128.735°**.
태양 고도는 위도만으로 정해집니다.

| | 계산식 | 값 |
|---|---|---|
| 동지 정오 고도 | 90 − 35.845 − 23.44 | **약 30.7°** |
| 하지 정오 고도 | 90 − 35.845 + 23.44 | **약 77.6°** |

### 세대 크기·위치는 호수마다 실측값

주택형이 다르면 세대 크기도 창 위치도 달라지고, 그래서 **조망 시점도 호수마다 달라집니다.**
배치도에서 날개(세대)를 범례 색상으로 하나씩 분리해 호수별로 직접 measured 했습니다
(`js/unitData.js`). 36개 세대 중 **33개가 배치도 색상과 동·호수 배치도의 주택형이 정확히 일치**했고,
동 번호 배지에 가려 측정이 잘린 2개만 주택형 표준치수로 대체했습니다.

| 예시 | 창면까지 거리 | 전면 폭 |
|---|---|---|
| 104동 2호 (152㎡A) | 25.3m | **13.8m** |
| 104동 3호 (132㎡B) | 28.7m | 11.4m |
| 109동 1호 (115㎡A) | 24.4m | 9.6m |
| 109동 4호 (84㎡A) | 21.3m | 8.6m |

전면 폭이 넓을수록 창이 크므로 조망 시야도 그만큼 넓게 트입니다.

### 🎨 평형별 색상

**배치도 범례와 똑같은 색을 3D 동의 세대(날개)에 그대로 입혔습니다.**
어느 블록이 어떤 평형인지 3D에서 바로 보이고, 옥상 색은 배치도를 위에서 본 것과 일치합니다.

| 색 | 주택형 | 있는 동 |
|---|---|---|
| 남색 / 연하늘 | 84㎡A / 84㎡B | 105~109 |
| 진초록 / 민트 | 115㎡A / 115㎡B | 105~109 |
| 주황 / 연주황 | 123㎡A / 123㎡B | 101~103 |
| 갈색 | 132㎡A / 132㎡B | 104 |
| 자주 / 연분홍 | 137㎡B / 137㎡C | 101~103 |
| 빨강 / 연빨강 | 152㎡A / 152㎡B | 104 |

- 패널의 색상표 항목을 누르면 **그 평형이 있는 동·호수로 바로 이동**합니다.
- 호수를 고르면 해당 **날개가 노란 테두리로 강조**되어 어디인지 바로 보입니다.
- `동에 평형별 색상 입히기` 체크를 끄면 원래 외벽색으로 돌아갑니다.
- **2차 동은 주택형이 공개되지 않은 추정값**이라 색을 입히지 않고 중립색으로 뒀습니다.

### 2차(남측) 9개 동

층수가 공개되지 않아 세대수로 역산했습니다.

```
총 3,443세대 − 1차 1,712세대 = 1,731세대
9개 동 × 4라인 = 36라인 → 1,731 ÷ 36 ≈ 48세대/라인 → 약 48층
```

2차는 1차 남쪽에 있어 **104·105동의 남향 일조와 조망에 직접 영향**을 줍니다.
패널의 `2차 층수` 슬라이더로 값을 바꿔가며 확인하세요.

## 화면 사용법

### ☀️ 날짜 · 시각
- **하지 / 춘분 / 추분 / 동지** 버튼. 일조 검토는 해가 가장 낮은 **동지**가 기준입니다.
- `▶ 하루 시간 흐름 재생` — 그림자가 하루 동안 어떻게 도는지 애니메이션.

### 🏢 세대 선택
- **동 / 호수 / 층** 선택 (호수 목록에 주택형이 함께 표시됩니다)
- `👁 조망 보기` — **그 세대 거실 창가(눈높이 1.62m)** 로 카메라가 들어갑니다.
  화면 가장자리에 그 세대 전면 폭에 맞춘 **창틀과 발코니 난간**이 보여서,
  드론 시점이 아니라 실제로 창밖을 내다보는 느낌으로 확인할 수 있습니다.
  하단에 평형 색상 칩 · 지상 높이 · 전면 폭 · **정면 앞 건물까지 거리(개방감)** 가 함께 표시됩니다.
- `📊 일조 분석` — 그날 하루 일조 시간 계산
- `⚖️ 이 층 1~4호 한번에 비교` — 같은 층 네 호수를 한 표로 비교

### 📊 결과 읽는 법

| 항목 | 의미 |
|---|---|
| 총 일조 | 하루 중 직사광이 드는 총 시간 |
| 연속 최대 | 끊기지 않고 이어지는 최대 일조 시간 |

## 원리 요약

**일조 계산**은 원리가 단순합니다.

1. 선택한 세대의 창문 위치를 3D 공간에서 찾는다
2. 그 지점에서 **태양 방향으로 광선을 하나 쏜다**
3. 가는 길에 다른 동이 걸리면 → 그 시각엔 해가 안 든다
4. 일출부터 일몰까지 **5분 간격으로 반복**해서 합치면 하루 일조 시간

태양 방향은 `sun.js`가 NOAA 천문 공식으로 날짜·시각·위도·경도에서 계산합니다.

## 파일 구조

```
3d_apartment/
├── index.html            화면 구조 (조작 패널)
├── css/style.css         스타일 (모바일 대응)
├── assets/
│   ├── db.jpg            공식 단지 배치도 (좌표 추출 원본 · 검증용 오버레이)
│   └── terrain.png       배치도를 픽셀 분류해 다시 그린 지형 (바닥에 깔림)
└── js/
    ├── siteData.js       ★ 단지 데이터 (동 좌표·층수·주택형·축척·정북)
    ├── unitData.js       호수별 실측 치수 (창면 거리·전면 폭·평형 색상)
    ├── sceneryData.js    자동 추출된 수목 1,678그루 + 저층 블록
    ├── sun.js            태양 위치 계산 (NOAA 천문 공식)
    ├── buildings.js      십자형 동 생성 — 주택형별 날개 치수 적용
    ├── site.js           지형·호수·수목·원경·나침반
    ├── analysis.js       일조 분석(광선 추적) + 결과 그래프
    └── main.js           전체 조립 + UI + 렌더 루프
```

## 값을 고치고 싶다면

`js/siteData.js` 한 파일만 보면 됩니다.

```js
export const PX_TO_M = 0.7111;        // 축척 (도로 폭 20m 실측)
export const NORTH_OFFSET_DEG = 24;   // 정북 (나침반 실측)

// 주택형별 세대 치수 — 여기를 바꾸면 동 크기가 바뀐다
export const UNIT_TYPES = { '152A': { area: 152, w: 11.6, len: 19.7 }, ... };

// 동 데이터 — px는 배치도(db.jpg) 위 십자 중심 좌표
{ id: '104', px: [790.57, 720.14], axis: 4.86, core: 14.0, floors: 59, line1: 180,
  types: ['152B', '152A', '132B', '132A'] },
```

패널의 `원본 배치도 겹쳐 보기`를 켜면 원본 배치도가 반투명하게 깔려서
3D 동이 배치도 십자 위에 정확히 앉았는지 바로 확인할 수 있습니다.

## 이 모델의 한계

- **1차 9개 동**은 배치도 픽셀 분석 기반이라 위치·방향이 정확합니다.
- **2차 9개 동**은 십자 중심을 눈으로 읽은 근사값이고, 층수는 세대수 역산 추정치입니다.
- 층고는 2.9m로 일괄 적용했습니다 (실제 층고가 확인되면 `FLOOR_HEIGHT` 수정).
- 주변 산 지형은 실제 지형도가 아니라 원경 분위기용입니다.
  실제 원경 조망까지 검토하려면 국토지리정보원 수치표고모델(DEM)을 넣어야 합니다.
